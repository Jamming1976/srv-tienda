🤖 Opción 2 — Script Python de actualización automática

Este script consulta la API de ML para cada producto en tu catálogo, actualiza los precios base y recalcula con tu markup. Correlo manualmente o programalo con cron.

import requests, json MARKUP = 0.30 # 30% CATALOG_FILE = "catalog.json" def get_ml_price(item_id): url = f"https://api.mercadolibre.com/items/{item_id}" r = requests.get(url) return r.json().get("price", 0) with open(CATALOG_FILE) as f: catalog = json.load(f) for p in catalog: new_base = get_ml_price(p["ml_id"]) if new_base and new_base != p["price_ml"]: print(f"Actualizando {p['title']}: ${p['price_ml']} → ${new_base}") p["price_ml"] = new_base p["price_srv"] = round(new_base * (1 + MARKUP), 2) with open(CATALOG_FILE, "w") as f: json.dump(catalog, f, indent=2) print("✅ Catálogo actualizado")

⏱️ Opción 3 — Actualización en tiempo real (en la tienda)

📊 Opción 4 — Google Sheets como backend

Con Google Apps Script podés crear una hoja que actúe como CMS: modificás precios, títulos y visibilidad desde Sheets y la tienda los lee automáticamente. Ideal si manejás el catálogo con tu equipo.

// En Google Apps Script → Crear un endpoint GET: function doGet() { const sheet = SpreadsheetApp.getActiveSpreadsheet() .getSheetByName("Catalogo"); const rows = sheet.getDataRange().getValues(); const [headers, ...data] = rows; const json = data.map(row => Object.fromEntries(headers.map((h, i) => [h, row[i]])) ); return ContentService .createTextOutput(JSON.stringify(json)) .setMimeType(ContentService.MimeType.JSON); }
